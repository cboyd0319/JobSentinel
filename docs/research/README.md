# Research And Evaluation

JobSentinel research supports one goal: help job seekers find real, relevant,
fairly compensated work while protecting sensitive job-search data.

## Principles

- Synthetic-data-first: use synthetic resumes, salary floors, candidate
  profiles, suspicious postings, and adversarial examples by default.
- Public-postings-only default: use public official job postings and public ATS
  postings unless a user has given explicit informed consent.
- Local-first evaluation: do not centralize real job-search histories, resumes,
  salary floors, notes, or application records.
- Candidate-side framing: evaluate whether JobSentinel helps users understand
  role fit, stale-posting risk, compensation transparency, and application
  readability.
- Clear limits: publish methodology, evaluation notes, limitations, and
  responsible-use guidance with research outputs.

## Allowed By Default

- Public official job postings.
- Public ATS postings.
- Synthetic resumes.
- Synthetic salary floors.
- Synthetic candidate profiles.
- Synthetic suspicious, scam, or adversarial postings.
- Aggregate evaluation metrics.

## Requires Explicit Informed Consent

- Real user resumes.
- Real application history.
- Private notes.
- Salary floors tied to a real person.
- Demographic-linked outcomes.
- Contact lists.
- Recruiter communications.
- Any PII-heavy dataset.

## Expected Outputs

Research outputs should make the work inspectable:

- Research question and scope.
- Dataset description, including whether data is public, synthetic, or
  consented.
- Evaluation method and known limits.
- Privacy and responsible-use notes.
- Product implications for ghost-job detection, salary transparency,
  pay-equity support, ATS transparency, and long-term unemployment support.
