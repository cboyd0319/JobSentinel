# JobSentinel v1.4.0 Release Notes

**Release Date:** January 2026
**Codename:** Ghost Hunter

## Highlights

Version 1.4 introduces **Ghost Job Detection** - an intelligent system that identifies fake,
stale, or already-filled job postings so you can focus on real opportunities. Additionally,
all user data has been migrated from localStorage to SQLite for true persistence.

## New Features

### Ghost Job Detection (E1)

JobSentinel now analyzes every job posting for signs it may be a "ghost job":

- **Stale Detection**: Flags jobs posted 60+ days ago
- **Repost Tracking**: Identifies frequently reposted positions
- **Content Analysis**: Detects vague descriptions and generic phrases
- **Company Analysis**: Flags companies with excessive open positions
- **Requirement Analysis**: Identifies unrealistic experience requirements

Each job receives a ghost score from 0.0 (definitely real) to 1.0 (likely ghost).

### Data Insights & Metrics (E2)

- **Score Breakdown Tooltip**: Hover over scores to see factor breakdown
- **Application Conversion Stats**: Quick stats bar on Applications page
- **Resume Match Visualization**: Green/red skill match indicators

### Backend Persistence (E3)

All user data now persists in SQLite instead of browser localStorage:

- **Cover Letter Templates**: Full CRUD with categories (General, Tech, Creative, Finance, Healthcare, Sales, Custom)
- **Interview Prep Checklists**: Per-interview completion tracking stored in database
- **Interview Follow-up Reminders**: Thank-you note tracking persisted
- **Saved Search Filters**: Filter presets stored with last-used timestamps
- **Search History**: Unlimited history (no more 10-item localStorage cap)
- **Notification Preferences**: Advanced filtering preferences persisted

**Migration**: Existing localStorage data is automatically migrated on first launch.

### Dashboard Enhancements

- **Ghost Filter**: New dropdown to show all jobs, real jobs only, or ghost jobs only
- **Ghost Indicators**: Visual badges on job cards showing ghost probability
- **Tooltips**: Hover to see specific reasons why a job was flagged

### UI Polish (E4)

- **Cover Letter Auto-Fill**: "Use for Job" button pre-fills template placeholders
  (`{company}`, `{position}`, `{location}`, `{date}`) from selected job
- **Keyboard Shortcut Badges**: Reusable `ShortcutKey` component for visual hints on buttons
- **Tour Integration**: "Take a guided tour" link in keyboard shortcuts help modal
  (press `?`)

### New Database Tables

- `job_repost_history` - Tracks job reposting patterns over time
- `cover_letter_templates` - User cover letter templates with categories
- `interview_prep_checklists` - Per-interview prep item completion
- `interview_followups` - Thank-you note tracking
- `saved_searches` - Filter presets with usage tracking
- `search_history` - Unlimited search history
- `notification_preferences` - JSON-stored notification config

### New API Commands (23 total)

Ghost Detection (3):

- `get_ghost_jobs` - Retrieve flagged ghost jobs
- `get_ghost_statistics` - Get aggregate detection stats
- `get_recent_jobs_filtered` - Filter jobs by ghost status

User Data (20):

- Templates: `list_cover_letter_templates`, `get_cover_letter_template`,
  `create_cover_letter_template`, `update_cover_letter_template`,
  `delete_cover_letter_template`
- Interview Prep: `get_interview_prep_checklist`, `save_interview_prep_item`
- Follow-ups: `get_interview_followup`, `save_interview_followup`
- Saved Searches: `list_saved_searches`, `create_saved_search`, `use_saved_search`, `delete_saved_search`
- Notifications: `get_notification_preferences`, `save_notification_preferences`
- History: `add_search_history`, `get_search_history`, `clear_search_history`
- Migration: `import_cover_letter_templates`, `import_saved_searches`

## Known Limitations

1. Ghost detection accuracy improves with historical data
2. Large staffing agencies may have higher false positive rates
3. Some legitimate long-running searches may be flagged
