# JobSentinel v2.9.1 Release Notes

Status: release-notes snapshot prepared for the `2.9.1` maintenance release.
Use the official GitHub Releases page to confirm the currently published latest
release and assets.

JobSentinel `2.9.1` is a maintenance and repo cleanup release. It preserves the
`2.9.0` user-facing feature set while improving verification, dependency
currency, repo organization, and release-state accuracy.

## Before Installing Or Updating

- Use the official GitHub Releases page for downloads.
- Back up Settings before replacing the app.
- Verify the matching `.sha256` checksum before opening a package or installer.
- Expect `_unsigned` in Windows installer filenames when Windows signing
  credentials are unavailable.
- Expect `_no-account_` and `_universal` in macOS DMG filenames until Developer
  ID signing and notarization are available.
- JobSentinel does not silently update itself.

## What Changed

- Refreshed stable direct npm and Rust crate pins.
- Kept package, Tauri, and Cargo metadata aligned at `2.9.1`.
- Split oversized or near-budget maintained files so tests, mocks, architecture
  docs, and source-health checks stay easier to review.
- Replaced custom lazy statics with standard lazy initialization where safe.
- Narrowed safe Clippy allowances and removed obsolete file-size exceptions.
- Updated maintained docs so `v2.9.1` is clearly a cleanup line and historical
  `v2.9.0` evidence stays historical.
- Hardened app-wide keyboard shortcut startup timing so keyboard help, command
  palette, and navigation shortcuts are ready with the visible app shell.
- Added a v2.9.1 regression validation ledger with local browser, frontend,
  backend, build, docs, bloat, harness, and isolated macOS native startup
  evidence.

## What Did Not Change

- No new product features.
- No new source adapters.
- No new external AI features or provider transports.
- No new browser automation or restricted-source behavior.
- No change to local-first storage, credential safety, explicit user review, or
  optional external AI defaults.

## Validation Summary

Local regression evidence on 2026-06-22 covered:

- Full Playwright browser regression budget: 278 expected, 0 unexpected, 0
  flaky, 0 skipped.
- Docs screenshots: 9 Chromium captures with no tracked screenshot drift.
- Frontend unit tests: 3261 passing tests.
- Production frontend build.
- Rust library tests: 2958 passing tests, 11 ignored.
- Isolated macOS native startup through `tauri dev`, including encrypted local
  database creation and owner-only smoke-root permissions.
- Harness, docs lint, repo bloat, release version, release readiness, Clippy,
  and Rust formatting gates from the cleanup line.
- Final repo-wide pass caught and fixed a startup keyboard shortcut timing race;
  the focused Chromium/WebKit keyboard E2E and full browser regression budget
  passed after the fix.

Windows, Linux, public release assets, live restricted-source account sessions,
and credentialed external notification or external AI provider sends were not
claimed in the local cleanup validation pass.

## Platform Notes

- Windows 11+ remains a primary target. Signed installers are used when
  credentials exist; no-account builds are clearly labeled `_unsigned`.
- macOS remains a primary target. No-account universal DMGs are not Developer ID
  signed or notarized until Apple Developer Program materials exist.
- Linux packages remain part of the cross-platform release path.
- Agent Skills ZIP and tar.gz archives are packaged separately from desktop
  installers and should ship with matching checksums.
