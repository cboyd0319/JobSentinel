# v2.5.3 Release Notes - LinkedIn Auto-Connect

> **Release Date:** 2026-01-24
> **Type:** Feature Release
> **Breaking Changes:** None

---

## Overview

JobSentinel v2.5.3 introduces **LinkedIn Auto-Connect**, eliminating the need for any technical
knowledge to connect your LinkedIn account. No more copying cookies from DevTools – just click,
log in, and you're done.

---

## New Features

### LinkedIn Auto-Connect (macOS)

**Before (v2.5.2 and earlier):**

1. Log into LinkedIn in browser
2. Open DevTools (F12)
3. Navigate to Application → Cookies
4. Find and copy the `li_at` cookie
5. Paste into JobSentinel Settings

**After (v2.5.3):**

1. Click "Connect LinkedIn"
2. Log in normally (username, password, 2FA)
3. Done!

### How It Works

1. **Native WebView Window**: Opens a secure login window using Tauri's `WebviewWindowBuilder`
2. **Navigation Monitoring**: Detects successful login by URL pattern matching (`/feed`, `/jobs`, `/mynetwork`, etc.)
3. **Native Cookie Extraction**: Uses macOS `WKHTTPCookieStore` via objc2 to extract the `li_at`
   cookie directly from the system cookie store
4. **Secure Storage**: Cookie stored in macOS Keychain (never in plaintext config files)

## Technical Details

### New Dependencies (macOS only)

```toml
[target.'cfg(target_os = "macos")'.dependencies]
objc2 = "0.5"
objc2-foundation = { version = "0.2", features = ["NSString", "NSArray", "NSDictionary", "NSDate", "NSURL", "block2"] }
objc2-web-kit = { version = "0.2", features = ["WKWebsiteDataStore", "WKHTTPCookieStore", "WKWebViewConfiguration", "WKWebView", "block2"] }
block2 = "0.5"
```

## Security Considerations

- **Native Cookie Store Access**: Uses official Apple `WKHTTPCookieStore` API
- **No Password Storage**: Only session cookie is stored, never credentials
- **OS Keychain**: Cookie stored in macOS Keychain with app-scoped access
- **HttpOnly Respected**: Native extraction required because JavaScript cannot access HttpOnly cookies

## Migration

No migration required. Existing LinkedIn connections continue to work. Users can optionally
re-connect using the new auto-connect feature.
