# JobSentinel v2.1.0 Release Notes

**Release Date:** January 17, 2026

## Job Source Status

This historical release added comprehensive health monitoring for job-source
adapters, including a dashboard, retry logic, and credential expiry tracking.
Current source policy treats LinkedIn as a user-opened search-link destination
only.

### Highlights

- **Source adapters wired** - Scheduled source adapters connected to the scheduler
- **Source Status Dashboard** - Check job source status from Settings > Troubleshooting
- **Automatic Retry** - Exponential backoff on transient failures (429, 5xx errors)
- **Source Checks** - Verify source availability without a full search
- **Credential Expiry** - Warning support for eligible external-channel credentials

---

## New Features

### Job Source Status Dashboard

Access from: **Settings > Troubleshooting > "View Job Sources"**

| Feature | Description |
|---------|-------------|
| Summary Stats | Counts for working, having trouble, not working, and off sources |
| Source Table | Status, recent success, average check time, jobs found |
| Check History | Click any source to view recent check history |
| Source Checks | Check individual or all sources with one click |
| Connection Warnings | Alerts for supported expiring external-channel credentials |

### Source Adapter Expansion

This historical release wired additional source adapters into the scheduler.
Current source policy may disable or convert specific destinations to
user-opened search links when background access is not appropriate.

**Newly connected (8):**

- RemoteOK
- Wellfound (AngelList)
- WeWorkRemotely
- BuiltIn
- HN Who's Hiring
- Dice
- YC Startup Jobs
- ZipRecruiter

**Previously connected (5):**

- Greenhouse
- Lever
- LinkedIn
- Indeed
- JobsWithGPT

### Exponential Backoff Retry Logic

Source adapters automatically retry on transient failures:

```text
Default Configuration:
- Max attempts: 3
- Initial delay: 1 second
- Max delay: 30 seconds
- Backoff multiplier: 2.0x
- Retryable codes: 429, 500, 502, 503, 504
```

### Credential Health Tracking

Credential expiry tracking now supports user-configured external-channel
credentials where applicable. Current source policy does not collect LinkedIn
session credentials.
