# Commercial Superiority Bar

V3 should aim to be better than commercially similar products in almost every
way that matters to a job seeker. The goal is not feature mimicry. The goal is a
better outcome for the user: less manual work, better evidence, safer decisions,
more useful source coverage, stronger privacy, and clearer next actions.

This bar is paired with
[Job Seeker Platform Research](job-seeker-platform-research.md). The benchmark
must prove practical advantage for a free user, not only feature parity with
paid products.

## Product Classes To Beat

Use these classes as benchmark targets:

| Product class | Examples | Common strengths | V3 should beat them by |
| --- | --- | --- | --- |
| Job-search trackers | Teal, Huntr, Jobscan tracker | Kanban, notes, contacts, browser save, reminders. | Turning jobs into case files with source lineage, risk, fit, resume evidence, follow-up, interview, offer, and outcome history. |
| Resume scanners and builders | Jobscan, Rezi, Resume Worded, Teal | ATS checks, keyword suggestions, match scores, resume templates. | Requirement-level evidence mapping, truthful bullet repair, Qwen3 reranking, hard-requirement blockers, ATS readability lab, and provenance for every suggestion. |
| Browser autofill and copilot extensions | Simplify, Huntr extension, Teal extension | Save jobs, autofill forms, track applications from browser pages. | User-visible browser companion, local ledger actions, review queue, restricted-source boundaries, one-click applied logging, source provenance, and no hidden session storage. |
| Auto-apply platforms | LoopCV, Sonara-style tools | High-volume matching and automated submissions. | Safer user-controlled apply cockpit, application packet prep, form rehearsal, one-click logging, quality-over-volume strategy, and no final submission without the user. |
| Job boards and aggregators | LinkedIn, Indeed, BuiltIn, Google Jobs, regional boards | Large job inventory and search filters. | Source graph across official APIs, employer careers pages, regional packs, public structured data, duplicate detection, ghost-job risk, and local user-owned records. |
| Employer research and salary sites | Company review sites, salary tools, public registries | Company context, pay signals, reviews, and public facts. | Employer dossiers with official-source provenance, user-owned outcomes, public registry signals, pay clarity, scam checks, and no central harvesting of private user history. |
| AI chat workflows | General chatbots and resume prompts | Flexible drafting and brainstorming. | Structured local workflows, privacy labels, Qwen3 evidence engine, external AI preview, deterministic fallbacks, receipts, and eval-tested outputs. |

## V3 Differentiators

V3 should be meaningfully better on these axes:

| Axis | V3 bar |
| --- | --- |
| Privacy | Local-first, no hosted account required, no telemetry requirement, no hidden resume upload, no restricted-source session storage. |
| Evidence | Every match, gap, blocker, resume suggestion, and recommendation cites local evidence or says what is missing. |
| Source breadth | Official APIs, public ATS pages, employer careers discovery, regional packs, browser companion, source health, and source policy classes. |
| Employer intelligence | Candidate-side employer dossiers with official domains, ATS links, pay clarity, public registry signals, user-owned outcomes, and sourced questions to ask. |
| Browser workflow | Save, import, log applied, track, note, compare, and risk-check from a visible page with local review and no hidden source access. |
| Resume quality | Requirement-by-requirement evidence mapping instead of generic keyword stuffing or a single opaque score. |
| Matching quality | Qwen3 embedding plus Qwen3 reranking, lexical grounding, skill taxonomy, hard blockers, hard negatives, and eval fixtures. |
| Automation | Automatic where safe, user-controlled where sensitive, and always visible for restricted sources or external side effects. |
| Security | Hiring firewall, scam workflow, prompt-injection guard, URL token scrubbing, source-pack quarantine, and privacy receipts. |
| Recovery | Backup, rollback, repair, safe support reports, model doctor, source doctor, Security Doctor, and plain-language failure states. |
| Accessibility | Keyboard, screen reader, high zoom, low-distraction, overwhelmed-user, and nontechnical-first workflows as release criteria. |
| Hardware access | Essentials and Portable paths that remain useful without large model downloads or high-end hardware. |
| Regional support | UK, EU, and India starter framework with taxonomy, pay, CV, location, and source-pack foundations. |
| Portability | Export, backup, no lock-in, readable support reports, pack uninstall, model cleanup, and supported v3 rollback. |
| Explainability | Debuggable dense, lexical, skill, reranker, blocker, source, salary, and location signals. |

## Feature Bar By Workflow

### Find Jobs

Commercial trackers usually rely on user saves, extensions, or a limited set of
job boards. V3 should combine:

- official-source checks
- company careers discovery
- public structured data extraction
- source graph classification
- regional packs
- browser companion visible capture
- duplicate and repost lineage
- source-health scores
- source policy receipts

Success measure:

- A nontechnical user can find and review useful roles without knowing which ATS
  or source technology a company uses.

### Decide Whether To Apply

Commercial tools often show match scores, keywords, or generic recommendations.
V3 should show:

- employer official-source context
- user history with the employer
- hard requirement coverage
- preferred requirement coverage
- concrete resume evidence
- missing evidence
- blockers
- salary and location fit
- source risk
- ghost-job and stale-posting risk
- "why this matched" and "why not this job"

Success measure:

- Users can make a decision faster without trusting an unexplained score.

### Research Employers

Commercial products often split employer context across paid network insights,
salary databases, review sites, and manual web searching. V3 should provide:

- employer identity and official-domain checks
- official posting and ATS source links
- public registry signals with dates and jurisdiction
- pay clarity and range-quality review
- user-owned application, interview, and offer history
- scam and impersonation safe next steps
- recruiter and interviewer questions based on sourced uncertainty

Success measure:

- A nontechnical user can understand whether an employer is worth deeper effort
  without paying for proprietary databases or uploading private history.

### Prepare Application Materials

Resume tools often optimize keywords and formatting. V3 should go further:

- local evidence graph
- truthful bullet repair
- requirement-level tailoring
- ATS readability lab
- screening answer provenance
- application packet builder
- exact resume version tracking
- unsupported keyword edit warnings

Success measure:

- Every application packet is stronger, truthful, and traceable to evidence.

### Apply And Track

Autofill tools reduce typing. V3 should reduce more of the whole workflow:

- one-click visible-page case-file creation
- one-click applied logging
- local form rehearsal
- attachment checklist
- follow-up creation
- tracker timeline
- post-apply risk and ghost-job review
- application outcome learning

Success measure:

- Applying to a job leaves behind a clean local record without spreadsheet work.

### Interview And Offer

Many trackers stop at "interview" or "offer" status. V3 should include:

- interview story bank
- role-specific prep
- company and source context
- post-interview debrief
- offer cockpit
- written-versus-verbal separation
- total-comp and cost review
- deadline pressure review
- counter, accept, and decline packet prep

Success measure:

- The app remains useful after the user gets an interview, not only before.

## Non-Negotiable Ways Not To Compete

V3 should not try to win by:

- hidden restricted-source scraping
- stored restricted-source cookies or tokens
- background account-backed refresh
- final application submission without the user
- fake resume claims
- keyword stuffing
- dark-pattern urgency or fear copy
- opaque hiring-probability claims
- selling user data or making telemetry required

When a commercial product offers high-volume application submission, JobSentinel should compete
with safer speed: better targeting, better packets, form rehearsal, faster
logging, and explicit user control.

## Benchmark Metrics

Track these against commercial alternatives:

- monthly cost to complete a serious search
- number of workflows blocked by paid tiers or account creation
- time from install to first useful action
- clicks to save a visible job
- clicks to log applied
- copy/paste steps per application
- percent of recommendations with evidence
- hard-blocker false positives
- duplicate and repost detection rate
- source coverage by source class and region
- employer context coverage with source provenance
- resume suggestions with provenance
- private data sent outside the device by default
- successful recovery from failed source, model, update, or vault state
- novice-user task completion rate
- modest-hardware responsiveness
- cross-platform feature parity
- export and rollback success

## Leveling-The-Field Bar

V3 should be judged by whether a user without extra money can still get:

- useful applicant-fit insight without a paid network subscription
- advanced resume and requirement review without scan limits
- tracking and follow-up without spreadsheet work
- scam, stale-posting, and source-confidence help without uploading private data
- salary and cost-of-work clarity without trading privacy for salary databases
- employer intelligence without paying for proprietary review or salary
  databases
- guided outreach and referral planning without proprietary recruiter access
- recovery, export, and backup without a hosted account

Where a commercial platform has proprietary data JobSentinel cannot access,
JobSentinel should provide the best legitimate local alternative and label the
gap honestly.

## Competitive Demo Script

V3 should be able to demo this sequence:

1. Install without creating an account.
2. Pick a region and target role.
3. Import a resume.
4. Search official and public sources.
5. Open a visible browser page and create a case file in one click.
6. See fit, risk, source, pay, duplicate, and ghost-job signals.
7. Open an employer dossier with official-source context and user-owned history.
8. Map every hard requirement to resume evidence or a gap.
9. Build an application packet.
10. Log applied and create follow-up reminders.
11. Prepare for interview.
12. Review an offer.
13. Export or back up all data.
14. Show privacy receipts proving what stayed local.

If v3 cannot make this flow feel simpler than commercial alternatives, it is not
ambitious enough.
