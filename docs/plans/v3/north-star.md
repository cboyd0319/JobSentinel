# V3 North Star

V3 should make JobSentinel feel like a private command center for finding work.
The user should not have to become a spreadsheet operator, source researcher,
ATS analyst, prompt engineer, resume optimizer, recruiter CRM admin, and salary
negotiation researcher just to find work.

## One Sentence

JobSentinel v3 is a local work-search operating system that turns user-approved
signals into clear next actions, better applications, stronger negotiation, and
less wasted effort.

## Product Promises

- **Find the real leads.** Surface jobs from official APIs, employer career
  pages, public boards, regional sources, user-opened pages, and imported leads.
- **Protect time.** Push stale, weak, scam-like, duplicate, low-fit, and
  low-pay roles out of the user's way.
- **Explain fit.** Tie every match, gap, blocker, and recommendation to
  concrete evidence.
- **Prepare better materials.** Build application packets from truthful local
  evidence, not invented claims.
- **Track the whole funnel.** Connect search lanes, source health, applications,
  interviews, follow-ups, offers, and outcomes.
- **Learn locally.** Improve recommendations from user actions and labels
  without telemetry or hidden cloud training.
- **Work beyond one market.** Make UK, EU, and India starter support possible
  through regional packs, taxonomies, CV profiles, and source manifests.
- **Support military transition.** Carry user-confirmed military service,
  occupations, credentials, and current clearance evidence into civilian
  resumes, matching, public-service paths, and application review without
  inventing equivalence or inferring protected veteran status or eligibility.
- **Use the OS intelligently.** Use native platform features and small
  on-device model APIs where they remove friction, while keeping JobSentinel's
  governed evidence engine in control.
- **Beat the category.** Aim to outperform commercial trackers, resume tools,
  browser copilots, and auto-apply products on user control, privacy, evidence,
  source breadth, recovery, and nontechnical ease.
- **Stay calm and usable.** Make advanced career campaign mechanics understandable
  to a person under pressure.
- **Install without friction.** Make the first useful workflow available with a
  small number of plain-language setup choices.

## Horizons

### Horizon 1: Integrated Job-Search Workspace

Make every current v2.9 surface feel connected:

- An installer and first-run flow that gets the user to useful work-search help
  without accounts, cloud setup, or model downloads.
- A dashboard that starts from what the user should do next.
- Opportunity case files that combine job details, fit, risk, source, resume,
  application, contacts, follow-ups, interviews, and offer notes.
- One-click local ledger events for saved, applied, rejected, follow-up,
  interview, offer, and not-interested states.
- Resume, application, and negotiation tools that read from the same evidence
  graph instead of isolated screens.

### Horizon 2: Source And Browser Expansion

Move beyond stock job boards:

- Source graph for official APIs, ATS tenants, employer pages, regional boards,
  professional associations, public-sector portals, and user-opened pages.
- Browser companion for visible, user-approved capture from pages the user is
  already viewing.
- Declarative source adapter packs with policy metadata, fixtures, and rate
  limits.
- Source health diagnostics so users know whether a source is fresh, blocked,
  stale, noisy, or still under review.

### Horizon 3: Local Job-Search Intelligence

Turn JobSentinel into a local assistant that plans with the user:

- Local agent runtime that can run skills, explain plans, and request approval.
- Qwen3 embedding plus reranker pipeline extended with labels, hard negatives,
  learning-to-rank, and counterfactual fairness tests.
- Local career graph that understands roles, skills, evidence, goals, source
  outcomes, pay floors, and user preferences.
- Privacy-preserving community packs for source adapters, search lanes, role
  taxonomies, and interview rubrics without uploading user data.
- Regional readiness packs for the UK, EU, and India so source, salary, CV,
  occupational taxonomy, and application-language differences do not stay
  hard-coded to the United States.

### Horizon 4: Full Operating-System Shift

If v3 requires a major refactor, the release should allow it:

- Move from page-centric UI to case-file and mission-centric workflows.
- Replace isolated feature tables with an event ledger plus derived views.
- Make all source, model, skill, and browser capabilities pack-driven with
  signed manifests.
- Treat local ML, agent skills, Browser Companion, and source adapters as
  pluggable runtimes with shared privacy receipts.
- Make the entire app observable through diagnostics, evals, and user-facing
  explanations.

## Wild But Aligned Outcomes

- The user opens JobSentinel and sees a daily mission list, not a blank tracker.
- The app explains why it hid a job, why a job is worth applying to, and what
  exact resume evidence supports the application.
- A browser side panel watches only the user-approved page and offers local
  actions such as Save, Log applied, Track, Compare resume, and Check risk.
- The app can say, "This role looks strong, but the required clearance blocks
  fit," instead of showing a single vague match score.
- A weekly review shows which sources created interviews, which lanes wasted
  time, and which resume version performed best.
- A pay review warns when a verbal number, written offer, location cost, or
  deadline pressure changes the real value of an offer.
- A local agent can prepare an application packet, but the user sees every
  field and submits outside JobSentinel.
- A novice user can install the app, connect no accounts, and still get useful
  job leads, resume checks, and follow-up help.
- A user on modest hardware can choose JobSentinel Essentials and still get the
  core search, review, resume, tracking, and pay-protection workflows without
  large model downloads.
- A user can recover from a failed update, missing model, source error, or
  browser companion issue with plain next steps instead of developer messages.
- A UK, EU, or India user can pick a starter region pack and see correct pay,
  location, CV format, and source-language assumptions even when full regional
  coverage is still marked as incomplete.
- A macOS user can ask a system shortcut to log a follow-up or summarize a
  selected visible posting, with the same local privacy receipt and review path
  as the desktop UI.
- A commercial-product comparison shows JobSentinel beating tracker, resume,
  browser-copilot, and auto-apply products on the workflows that matter without
  sacrificing privacy or user control.

## Anti-Goals

- Building an employer screening tool.
- Predicting whether an employer will hire the user.
- Turning private campaign data into telemetry.
- Requiring cloud AI for core value.
- Creating hidden scrapers for restricted authenticated sources.
- Automating final application submission.
- Encouraging fake resume claims, hidden keywords, prompt injection, or ATS
  manipulation.
- Exposing advanced model, source, or vector settings to novice users unless
  they explicitly choose expert mode.
- Treating OS-provided model output as authoritative when it has not passed
  JobSentinel's model governance, provenance, and eval requirements.
- Claiming full international coverage when v3 has only starter region packs
  and framework support.
- Chasing commercial application volume at the expense of application quality,
  user control, source boundaries, or truthful materials.
