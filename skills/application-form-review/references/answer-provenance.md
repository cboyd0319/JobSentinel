# Application Answer Provenance

Use this reference when application-form answers can affect screening,
privacy, legal status, pay, schedule, or resume consistency.

## Exact-Question Rule

Keep each answer tied to the exact prompt the user sees.

| Item | Required handling |
| ---- | ----------------- |
| Exact question | Preserve the wording before drafting an answer. |
| Proposed answer | Answer only the question asked. Do not broaden it. |
| Evidence | Name the resume, profile, record, job post, or user note that supports it. |
| Unknown | Mark missing facts as user questions, not assumptions. |
| User confirmation | Required for hard screening, sensitive, or voluntary fields. |

## High-Risk Question Types

| Topic | Risk | Safe handling |
| ----- | ---- | ------------- |
| Work authorization or sponsorship | Knockout and legal sensitivity | Use only user-confirmed status or ask. |
| Citizenship, clearance, background, drug screen, or age | Sensitive and sometimes jurisdiction-dependent | Explain the question and require user choice. |
| Salary history or expectations | Anchoring and privacy risk | Offer user-controlled wording or skip options when available. |
| Demographic, disability, or veteran status | Voluntary or protected context | Do not choose for the user. |
| Licenses, certifications, degree, language, or years of experience | Knockout and resume consistency | Match evidence exactly or ask. |
| Schedule, commute, travel, relocation, or physical demands | Practical must-have risk | Ask before committing flexibility. |

## Provenance Table

Use this table for disputed, repeated, or high-risk fields.

## Consistency Checks

Before the user submits, compare answers with:

- the resume version attached to the application;
- profile or saved-answer records the user chose to use;
- job-post hard requirements;
- prior submitted-claim notes for the same company or role family;
- salary floor or pay preference only when the user chose to share it.

If a form answer and resume claim disagree, pause and ask the user to choose
which source is correct before drafting final wording.

## Do Not Infer

Do not infer authorization, citizenship, sponsorship needs, protected answers,
salary flexibility, start date, commute tolerance, travel willingness,
certifications, clearances, degree status, language fluency, disability,
veteran status, background-check outcome, references, or permission to upload a
file.
