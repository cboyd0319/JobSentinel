# Resume Evidence Mapping

Use this when tailoring a resume to a specific role, especially when the job
post has hard requirements, dense keywords, or ambiguous screening language.

## Requirement Types

| Type | Action |
| ---- | ------ |
| Hard requirement | Verify first. If untrue, mark as skip, ask, or gap. Do not draft around it. |
| Required qualification | Map to visible evidence or ask for proof. Prioritize before preferences. |
| Preferred qualification | Add only when truthful and useful. Do not crowd out stronger evidence. |
| Role language | Use sparingly to improve readability and matching. Never keyword-stuff. |
| Unknown | Ask the user or leave out. |

## Evidence Strength

| Strength | Meaning | Resume action |
| -------- | ------- | ------------- |
| Strong | Recent, specific, visible proof with scope, tool, audience, or outcome | Use confidently |
| Partial | Related but older, smaller, adjacent, or not named in the resume | Use safer wording and ask for details |
| Unsupported | Plausible but not evidenced | Ask, do not add |
| Not true | User cannot defend it | Keep off the resume and form answers |

## Skill And Keyword Validation

Validate extracted skills before scoring fit or drafting edits.

| Item | Check | Action |
| ---- | ----- | ------ |
| Exact skill | Resume names the same tool, method, credential, or domain | Use if the context supports the job requirement |
| Adjacent or semantic skill | Resume uses related language, acronym, older name, or close method | Use safer wording and label as partial unless the user confirms |
| Technology-family match | Resume shows the broader family but not the exact tool | Mention only the family or ask whether exact exposure exists |
| False positive | Job title, employer, product slogan, department name, school name, or section heading was extracted as a skill | Remove and record the removal |
| Experience duration | Date ranges overlap or exposure was concurrent | Count overlapping time once and ask before using years |
| Academic, course, project, or volunteer exposure | Evidence exists outside paid professional work | Keep separate from professional experience unless the user chooses the wording |

Prefer structured job-post requirements when available. If the job post is a
free-text description, extract requirements into a reviewable list and ask the
user to confirm ambiguous must-haves before tailoring.

Use conservative equivalence only when terms are clearly the same credential,
concept, or constraint. Examples:

- `CRM` and `customer relationship management`
- `QA` and `quality assurance`
- `RN` and `Registered Nurse`; `CNA` and `Certified Nursing Assistant`
- `BLS` and `Basic Life Support`; `Security+` and `Security Plus`
- `CDL` and `commercial driver's license`
- `high school diploma`, `GED`, and `high school equivalency`
- `onsite`, `on-site`, and `on site`
- `work authorization` and `authorized to work`

Do not use broad fuzzy matching for credentials, licenses, authorization,
clearance, safety, or regulated work. Close wording is not enough when the
specific credential or legal status matters.

## Capability And Evidence Strength

Use the lowest defensible capability level unless the user confirms stronger
evidence:

| Level | Evidence pattern | Resume action |
| ----- | ---------------- | ------------- |
| Exposure | Observed, studied, read, or assisted | Use as learning or exposure only |
| Hands-on use | Performed tasks with guidance | State direct use, not ownership |
| Independent delivery | Delivered work without regular supervision | Use stronger action wording |
| Ownership | Managed, improved, or was accountable for workflow, system, or result | Use ownership wording with scope |
| Expert or strategic | Set standards, architecture, strategy, or organization-level direction | Use only with clear leadership evidence |

Evidence strength, strongest first:

1. metric-backed accomplishment;
2. scope-backed accomplishment;
3. ownership-backed work tied to a workflow, tool, system, team, or program;
4. clear responsibility with tool or method;
5. duty-only statement;
6. bare keyword.

Current or recent evidence can be more relevant than older evidence, but older
evidence still counts when the skill, credential, or domain remains current.

## ATS And Readability Checklist

- Standard headings: Summary, Experience, Skills, Education, Certifications, or
  equivalent plain-language sections.
- Normal selectable text, simple bullets, consistent dates, no hidden text, no
  images for critical content, and no table-heavy layout unless required.
- Role keywords appear only where evidence supports them.
- File name and visible contact details are professional and current.
- Summary and bullets are readable to a human, not only an ATS.

## Keyword Cap

For each important term, use one of:

- exact term if the candidate used it directly;
- adjacent truthful term if experience is related but not exact;
- question if evidence is missing;
- do-not-add if the claim would mislead.

Do not add a keyword more than once unless it improves a specific section.

## Bullet Quality

Prefer:

`Action + scope + method or tool + outcome`

Use metrics only when the user provides or approves them. If no metric exists,
use a truthful observable outcome such as standardized workflow, reduced manual
rework, improved handoff quality, clearer reporting, faster triage, or better
stakeholder visibility.
